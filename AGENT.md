# 新媒体运营文件框架 Agent 工作规则

> 适用范围：本目录下所有平台、账号与素材文件  
> 英文名：New Media Operation Document Framework  
> 项目名 / 简称：`nmo-df`  
> 最后更新：2026-05-15

## 2. 启动前必读

- 执行平台或账号相关任务前，先阅读对应平台的 `ACCOUNTS.md`。
- 涉及账号定位、目标人群、内容边界、负责人时，以对应平台的 `ACCOUNTS.md` 为准。
- 涉及价格、优惠、资质、合作条款、政策解释、效果承诺、项目介绍等敏感信息时，必须以最新官方口径为准；无法确认时不要自行补写为事实。

## 3. 目录用途

- `assets/`：全局素材库。存放跨平台可复用的图片、视频、封面、Logo、模板、设计源文件、活动素材等。
- `平台/ACCOUNTS.md`：平台账号档案。记录该平台所有运营账号、账号链接、负责人、定位和基础信息。
- `平台/账号/data_raw/`：账号原始运营数据。存放从平台后台、创作者中心、广告后台等手动导出的原始 Excel、CSV、ZIP 等文件。
- `平台/账号/exports/`：账号导出与中间文件。存放清洗后的表格、分析结果、自动化输出、待转存文件等。
- `平台/账号/automation/`：账号自动化相关。存放脚本、任务说明、配置、运行日志等。
- `平台/账号/archive/`：账号历史归档。存放旧数据、旧项目、已结束活动资料、废弃模板等。

## 4. 文件处理原则

- 不覆盖、修改或删除任何 `data_raw/` 中的原始数据文件。
- 数据分析、整理、清洗产生的新文件，不要放回 `data_raw/`，应放入对应账号的 `exports/`。
- 不删除 `assets/` 或 `archive/` 中的原始文件；如需清理，先生成清单并等待人工确认。
- 不在本地文件中记录账号密码、验证码、私钥、身份证号等敏感信息。
- 涉及个人姓名、肖像、联系方式、交易记录、评价反馈、案例数据、未公开业务信息时，先确认授权或仅做脱敏处理。

## 5. 自动化边界

- Agent 可以整理文件、生成分析、清洗数据、生成草稿、汇总复盘。
- Agent 不得自动发布内容、自动回复用户、自动删除素材、自动修改原始后台导出数据。
- 需要对外发布、对外回复、涉及承诺或政策解释的内容，必须交由人工确认。

## 6. 命名建议

- 日期使用 `YYYY-MM-DD` 或 `YYYY-MM`。
- 原始后台导出建议命名为：`YYYY-MM-DD_平台_账号_数据类型_原始导出.扩展名`。
- 清洗或分析结果建议命名为：`YYYY-MM-DD_平台_账号_数据类型_分析结果.扩展名`。
- 素材文件建议保留原始文件名；如需重命名，优先补充日期、账号、活动或栏目名称。

## 7. Notion 协作原则

- 本地 `ACCOUNTS.md` 保留账号核心上下文，不迁移到 Notion。
- Notion 适合承载选题库、发布记录、数据复盘、口径库、SOP、授权与合规记录、素材索引。
- 本地文件系统保留原始素材、后台导出原件、自动化脚本和账号级输出文件。
