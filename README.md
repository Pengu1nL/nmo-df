# 新媒体运营文件框架

**英文名：** New Media Operation Document Framework  
**项目名 / 简称：** `nmo-df`

这是一个用于新媒体运营工作的本地文件框架，适合管理多平台账号档案、运营数据、素材资产、自动化脚本和复盘输出。

模板采用“平台归档账号，账号归档数据，素材全局共享”的结构：平台目录负责账号上下文，账号目录负责数据与输出，`assets/` 负责跨平台素材沉淀。

## 快速开始

1. 复制整个 `nmo-df` 文件夹，作为新项目的运营文件框架。
2. 阅读 `AGENT.md`，确认文件处理规则和自动化边界。
3. 在对应平台的 `ACCOUNTS.md` 中填写账号总览、定位、人群、内容方向、运营目标和资料链接。
4. 将 `平台/账号名称/` 复制或重命名为实际账号名。
5. 按用途存放原始数据、分析结果、自动化脚本和历史归档。

## 账号目录用途

- `data_raw/`：存放平台后台、创作者中心、广告后台等导出的原始数据。不要覆盖、修改或删除这里的文件。
- `exports/`：存放清洗后的表格、分析结果、复盘报告、自动化输出和待转存文件。
- `automation/`：存放脚本、配置、任务说明、运行日志等自动化相关文件。
- `archive/`：存放旧数据、旧项目、已结束活动资料、废弃模板等历史内容。

## 素材目录用途

- `assets/brand/`：长期稳定的品牌资产，如 Logo、品牌色、字体、视觉规范、统一头像、品牌介绍图等。
- `assets/shared/`：可跨平台复用的通用素材，如场地、人物、服务场景、B-roll、音乐、通用模板等。
- `assets/campaigns/`：围绕一次活动、营销节点或专题推广产生的跨平台素材包。
- `assets/wechat/`：只服务于微信平台发布或排版的适配素材。
- `assets/xiaohongshu/`：只服务于小红书笔记、封面、图文卡片或短视频发布的适配素材。
- `assets/douyin/`：只服务于抖音短视频、封面、字幕或竖屏发布的适配素材。
- `assets/bilibili/`：只服务于 B站视频、封面、字幕、动态、专栏或合集发布的适配素材。

## 文件命名建议

- 日期统一使用 `YYYY-MM-DD` 或 `YYYY-MM`。
- 原始后台导出：`YYYY-MM-DD_平台_账号_数据类型_原始导出.扩展名`
- 清洗或分析结果：`YYYY-MM-DD_平台_账号_数据类型_分析结果.扩展名`
- 平台发布成品：`YYYY-MM-DD_账号_内容主题_用途.扩展名`
- 活动目录：`YYYY-MM-DD_活动名称`

## 工作边界

- 不在本地文件中记录账号密码、验证码、私钥、身份证号等敏感信息。
- 涉及个人姓名、肖像、联系方式、交易记录、评价反馈、案例数据、未公开业务信息时，先确认授权或做脱敏处理。
- 涉及价格、优惠、资质、合作条款、政策解释、效果承诺、项目介绍等敏感信息时，以最新官方口径为准。
- 可以用 Agent 整理文件、生成分析、清洗数据、生成草稿和汇总复盘。
- 不应让 Agent 自动发布内容、自动回复用户、删除素材或修改原始后台导出数据。

## 推荐配合 Notion 管理

本地文件夹适合存放原始素材、后台导出、脚本、账号级输出文件；Notion 更适合承载选题库、发布记录、数据复盘、口径库、SOP、授权与合规记录、素材索引。
