# 品牌资产存储规则

> 目录：`assets/brand/`  
> 最后更新：2026-05-15

## 1. 目录定位

`brand/` 用于存放长期稳定、可跨平台复用的品牌基础资产。

判断标准：如果一个素材代表“品牌身份”本身，而不是某次活动、某个平台或某条内容的专用素材，就优先放在这里。

## 2. 适合存放

- Logo、品牌标准字、品牌标识组合
- 品牌色、字体、VI规范、视觉规范文档
- 账号统一头像、统一背景图、通用片头片尾
- 品牌介绍图、产品或服务介绍图、长期使用的官方视觉素材
- 官方二维码、联系方式视觉模板
- 长期复用的品牌级设计源文件

## 3. 不适合存放

- 某次营销节点、线下活动、项目推广的活动素材
- 只适配小红书、抖音、微信某个平台尺寸的成品图
- 原始运营数据、后台导出文件
- 临时截图、未整理素材、废弃草稿

## 4. 建议子目录

```text
brand/
├── logo/        # Logo、品牌标准字、品牌标识组合、不同颜色版本
├── fonts/       # 授权字体文件、字体说明、字体使用规范
├── colors/      # 品牌色卡、配色方案、色值说明文件
├── vi/          # VI规范、品牌视觉手册、标准应用示例
├── avatars/     # 各平台账号头像、品牌头像、头像源文件
├── intro/       # 品牌介绍图、产品或服务介绍图、长期复用的官方介绍素材
└── templates/   # 品牌级模板、片头片尾、二维码版式、联系方式模板
```

## 5. 命名建议

- 品牌规范文件：`YYYY-MM-DD_品牌规范_版本说明.扩展名`
- Logo文件：`logo_用途_颜色版本_格式.扩展名`
- 头像文件：`平台_账号_头像_版本日期.扩展名`
- 模板文件：`用途_尺寸_版本日期.扩展名`

## 6. 使用原则

- 不轻易覆盖旧版品牌资产；新版应另存并标注日期或版本。
- 不确定是否为官方最终版时，在文件名中标注 `待确认`。
- 过期但仍可能追溯的版本，移动到对应子目录的 `archive/`，不要直接删除。
