# 活动与专题素材存储规则

> 目录：`assets/campaigns/`  
> 最后更新：2026-05-15

## 1. 目录定位

`campaigns/` 用于存放围绕某次活动、营销节点或专题推广产生的一整套跨平台素材。

判断标准：如果素材属于一次明确的 campaign，例如线下活动、营销节点、新品发布、专题栏目推广，就放在这里。

## 2. 适合存放

- 线下活动、营销节点、项目推广、品牌活动等活动素材
- 活动主视觉、海报、宣传图、短视频原片、活动照片
- 多平台共用的 campaign 封面、字幕条、贴纸、片头片尾
- 活动拍摄原片、精选素材、设计源文件
- 活动复盘需要引用的素材截图或素材包

## 3. 不适合存放

- 长期稳定的品牌资产，例如 Logo、VI规范、账号头像
- 与活动无关的通用场景素材
- 已经只为某个平台二次适配的最终发布成品
- 平台后台导出的数据文件

## 4. 存储结构

```text
campaigns/
├── YYYY-MM-DD_活动名称/
├── YYYY-MM-DD_活动名称/
└── YYYY-MM-DD_活动名称/
```

每次活动单独一个文件夹。活动文件夹内不强制拆分子目录，可直接存放该活动相关的照片、视频、海报、封面、源文件、成品和复盘素材。

如果活动素材较多，可以在活动文件夹内按需使用以下结构：

```text
YYYY-MM-DD_活动名称/
├── raw/        # 原始照片、视频原片、录音、未处理素材
├── design/     # 海报、封面、长图、设计源文件、排版文件
├── video/      # 成片、剪辑工程、字幕版视频、短视频切片
├── publish/    # 已用于发布的最终成品、各平台适配版本
└── review/     # 活动复盘截图、数据截图、复盘用素材
```

以上子目录是可选结构，不是强制要求。小活动可以不拆分，直接把文件放在活动文件夹内。

## 5. 命名建议

- 活动目录：`YYYY-MM-DD_活动名称`
- 原始素材：`YYYY-MM-DD_活动名称_素材类型_序号.扩展名`
- 成品素材：`活动名称_平台_用途_尺寸_版本.扩展名`

## 6. 使用原则

- 原片和源文件尽量保留，不覆盖。
- 平台适配后的成品可复制到 `assets/wechat/`、`assets/xiaohongshu/`、`assets/douyin/`。
- 活动结束后，不删除素材；保留在对应活动文件夹内，必要时在活动文件夹名称或文件名中标注版本。
