# 小红书素材存储规则

> 目录：`assets/xiaohongshu/`  
> 最后更新：2026-05-15

## 1. 目录定位

`xiaohongshu/` 用于存放只服务于小红书平台发布的素材成品与平台适配文件。

判断标准：如果素材已经按照小红书笔记、封面、图文卡片、短视频发布需求做了尺寸、标题、版式或导出适配，就放在这里。

## 2. 适合存放

- 小红书笔记封面
- 小红书图文卡片、长图、信息图
- 小红书短视频封面、字幕版成片
- 小红书账号主页截图、视觉参考截图
- 从通用素材或 campaign 素材二次适配后的小红书版本

## 3. 不适合存放

- 未经适配的拍摄原片、通用场景照片
- 品牌 Logo、VI规范、统一头像等品牌基础资产
- 微信或抖音专用成品
- 小红书后台导出的运营数据

## 4. 建议子目录

```text
xiaohongshu/
├── covers/       # 小红书笔记封面、视频封面、封面源文件
├── note-cards/   # 图文笔记卡片、信息图、长图切片、轮播图
├── videos/       # 小红书短视频成片、字幕版视频、口播切片
├── screenshots/  # 主页截图、笔记截图、评论截图、参考案例截图
├── templates/    # 小红书封面模板、图文卡片模板、标题版式模板
└── archive/      # 旧版封面、废弃设计、历史笔记素材归档
```

## 5. 命名建议

- 笔记封面：`YYYY-MM-DD_账号_笔记主题_封面.扩展名`
- 图文卡片：`YYYY-MM-DD_账号_笔记主题_卡片_序号.扩展名`
- 视频成片：`YYYY-MM-DD_账号_视频主题_小红书成片.扩展名`

## 6. 使用原则

- 小红书发布成品可以从 `brand/`、`shared/`、`campaigns/` 复制素材后再适配。
- 不在本目录存放后台原始数据；数据应进入对应账号的 `data_raw/`。
- 涉及人物肖像、用户评价、案例数据、作品或成果展示时，应确认授权或标注使用限制。
